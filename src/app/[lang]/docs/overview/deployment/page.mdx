# Deployment Guide

This guide walks you through deploying the application to different environments, including development, staging, and production.

## Prerequisites

- Ensure the application is tested and free of critical issues.
- Set up environment variables as described in the [Installation Guide](../installation/page.mdx).
- Have access to a server or cloud platform for deployment.

## Deployment Options

### 1. **Local Server**

1. Build the application:
   ```bash
   npm run build
   # or
   yarn build
   ```
2. Start the production server:
   ```bash
   npm run start
   # or
   yarn start
   ```
3. Access the application at `http://localhost:3000`.

### 2. **Docker**

1. Build the Docker image:
   ```bash
   docker build -t shadboard:latest .
   ```
2. Run the container:
   ```bash
   docker run -p 3000:3000 shadboard:latest
   ```

### 3. **Cloud Platforms**

#### Vercel

1. Connect your GitHub repository to Vercel.
2. Configure environment variables in the Vercel dashboard.
3. Deploy directly from the main branch.

#### AWS

1. Use AWS Elastic Beanstalk or EC2.
2. Upload the built application files.
3. Configure the server to run `npm start` after deployment.

#### Others

Refer to your specific hosting provider's documentation for detailed steps.

## Post-Deployment Checklist

- Verify the application is running as expected.
- Test all critical functionalities.
- Monitor logs for errors or warnings.

For further assistance, check the [Troubleshooting Guide](../troubleshooting/page.mdx).
